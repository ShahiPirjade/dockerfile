$ sudo yum install -y yum-utils

// yum-config-manager // It's a useful tool for administrators to easily manage software sources on a system using the YUM package manager. 

Step2 :  install and update Docker from the repository.
$ sudo yum install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
// This command installs Docker, but it doesn't start Docker. It also creates a docker group, however, 
it doesn't add any users to the group by default.


Step3 : Verify that the Docker Engine installation is successful by running the hello-world image.
$ sudo docker run hello-world

Step4 : Start Docker  
$ sudo systemctl start docker

$ sudo systemctl enable docker

How to access a nginx or any docker application running in VM? Ans->Port forwarding.
Every server/container has 0-65535 PORTS 
So there will be 2 ports. one at the VM level and the other at the Container level. 
So if any user is accessing container/application nginx. Ex - if a user hits an application 8081, 
We can inform docker to forward the request to nginx(which runs on port 80).
 
$ docker run -p <host-port>:<container-port> image                                                        [Interview Question - How do you do port forwarding?]
eg. docker run -p 8081:80 <nginx-image>--> runs in forground and attaches to screen, we cant run anything, so therefore we need to detach it. 
host-port can be anything but container-port will be fix                       
container-port is inbuild in image, if you want to change you need to change it in the image.
$ docker run -d -p 8081:80-->		[Interview Question - Detaches the screen and runs in background]

Dockerfile 
--------
*FROM FOLDER*
FROM centos:8  //While creating image you need to write which image you want.

*RUN FOLDER*
FROM centos:8
RUN yum install nginx -y  // on top of base OS we are installing nginx 

push the above changes from git and 

docker build -t run:v1 . // 

FROM --> to select base OS
RUN --> to install or configure packages or image, inside base OS it will run at image creation. 
		[Run is used to install or configure the packages.]
CMD --> it is to run the container, it will run at container creation.  there will be only 1 CMD instruction in a Dockerfile. If you list more than 1 CMD then only the last CMD will take effect.
		[It will execute at the time of docker run or container creation]
